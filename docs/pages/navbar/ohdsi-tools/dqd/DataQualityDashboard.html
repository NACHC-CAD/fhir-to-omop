Install, Run, and View The OHDHSI Data Quality Dashboard (DQD)

The instructions here will guide you through the installation of the OHDSI Data Quality Dashboard (DQD). The Data Quality Dashboard is a tool written in R that can perform a number of configurable tests on your OMOP instance. More information is available at https://github.com/OHDSI/DataQualityDashboard The DQD collects meta data for your instance by running an R script that generates a single JSON file. This file is then read as a web page using any browser. The instructions here show how to view your DQD information using a server that is launched from R.

Installing and Running DQD Quick Links

The scripts to in stall, run, and view the DQD are in Github here. Click on the links below to download the scripts individually.



  • install-dqd.r: Run the R script here to install components required for the DQD.
  • init-params-for-dtq.r: Run the R script here to initialize parameters to run either the DQD or to view the results.
  • run-dqd.r: Run the R script here to run the DQD.
  • view-dqd.r: Run the R script here to view the results.


Run the install-dqd.r script one time to install the components required to run or view the DQD.

Run the init-params-for-dtq.r script before you run either the run-dqd.r script or the view-dqd.r script. This script contains all of the parameters to run or view the DQD. This script can be edited to fit your OMOP instance. The other scripts should not need to be edited to run the scripts.

Run the run-dqd.r script to generate the metrics for your OMOP instance. This will take several minutes.

Run the view-dqd.r script to view the results. Runing this script will start the "shiny" server and should open the DQD web application in your default browser.
